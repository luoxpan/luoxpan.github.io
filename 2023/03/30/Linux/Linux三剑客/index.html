linux三剑客grep,sed,awk

正则表达式

元字符 描述
. 句号匹配任意单个字符除了换行符。
[ ] 字符种类。匹配方括号内的任意字符。
[^ ] 否定的字符种类。匹配除了方括号里的任意字符
* 匹配>=0个重复的在*号之前的字符。
+ 匹配>=1个重复的+号前的字符。
? 标记?之前的字符为可选.
{n,m} 匹配num个大括号之前的字符或字符集 (n <= num <= m).
(xyz) 字符集，匹配与 xyz 完全相等的字符串.
| 或运算符，匹配符号前或后的字符.
\ 转义字符,用于匹配一些保留的字符 [ ] ( ) { } . * + ? ^ $ \\ &#124;
^ 从开始行开始匹配.
$ 从末端开始匹配.

简写字符集

简写 描述
. 除换行符外的所有字符
匹配所有字母数字，等同于 [a-zA-Z0-9_]
匹配所有非字母数字，即符号，等同于： [^\w]
匹配数字： [0-9]
匹配非数字： [^\d]
匹配所有空格字符，等同于： [\t\n\f\r\p{Z}]
匹配所有非空格字符： [^\\s]
匹配一个换页符
匹配一个换行符
匹配一个回车符
匹配一个制表符
匹配一个垂直制表符

    掌握几个基础的就行。其它用到再查 ## awk 文本处理工具

  • 作用：处理文本，将文本按指定规则进行处理

  • 格式 awk [选项] '匹配规则和处理规则' [路径]

  • 选项

    • -F设置输入字段分隔符，也可以用处理规则中的BEGIN{}

  • 处理规则：

    • BEGIN{}处理文本之前设置变量
    • //使用的匹配规则，正则表达式或字符串
    • { }一条命令，可以是循环或判断
    • END{}处理文本之后执行END中的指令

  • awk中的变量

awk的处理逻辑其实很简单，首先执行BEGIN{}中的内容，只要是设置输入分隔符FS和输出分隔符OFS，执行/ /中的匹配规则，找出符合匹配的行，这些行逐行的执行后面的处理指令。最后执行END{}中的指令，主要是输出一些自定义的变量自定义值等。
